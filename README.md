# Personal Portfolio Website

此為用 React + Vite 開發的雙語系個人作品集網站。

## 部署與更新指南

如果未來有修改語言包 (`src/data.js`) 的資料或任何版面內容，請務必按照以下兩個步驟來「更新公開網站」與「備份原始碼」。

### 1. 更新公開網站內容 (Deploy)
這個指令會幫你將最新的程式碼過濾編譯打包，並且自動推送到 GitHub 的 `gh-pages` 分支。網頁伺服器會根據這個分支更新對外公開的那一頁。

```bash
npm run deploy
```
*(備註：因 GitHub Server cache 關係，每次執行完成後，大概等個 1~2 分鐘重新整理網頁即可看到最新畫面！)*

---

### 2. 備份原始碼至 GitHub 主分支 (Version Control)
為了確保你寫的心血與原始碼不會遺失，請將修改後的檔案也上傳進儲存庫的預設 `main` 分支當作紀錄。請在終端機依序輸入以下三個指令：

```bash
# 1. 追蹤所有變更的檔案
git add .

# 2. 建立提交紀錄 (引號內的文字可依這次的修改內容做調整)
git commit -m "update: 更新網站內容或履歷資料"

# 3. 將紀錄推送到遠端伺服器 (上傳)
git push
```

---

## 本機開發環境啟動
未來如果在不同電腦拉取這個專案，想要在本機預覽修改畫面：

```bash
# 1. 安裝所有開發環境依賴套件 (全新環境只需執行此行一次)
npm install

# 2. 啟動開發者伺服器
npm run dev
```
